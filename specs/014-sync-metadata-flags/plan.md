# Implementation Plan: Sync Script Metadata Flags + Consolidation

**Branch**: `014-sync-metadata-flags` | **Date**: 2026-04-25 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/014-sync-metadata-flags/spec.md`

## Summary

Consolidate the two near-identical sync scripts (`sync.sh`, `export-icloud.sh`) into a
single canonical `scripts/sync.sh`, add the missing osxphotos flags so favorite,
person, album, and sidecar metadata are written to the export, and add a manual-only
bats test suite that verifies each metadata field round-trips into the exported file.
After consolidation, run one full sync to backfill missing metadata into the existing
~80,258 exported files.

## Technical Context

**Language/Version**: Bash 3.2+ (macOS default; works without bash 4 features), Python 3.13 (already installed on Mac Mini for osxphotos)
**Primary Dependencies**: osxphotos 0.75.6 (Python), exiftool 13.55 (CLI), bats-core 1.10+ (already used in project per `002-immich-setup`)
**Storage**: Files on `/Volumes/HomeRAID/icloud-export/`; SQLite for tracking (`.osxphotos_export.db`, Photos.sqlite — read-only)
**Testing**: bats-core (consistent with existing test layer for `002-immich-setup`)
**Target Platform**: macOS Sequoia (Mac Mini); macOS-only (osxphotos depends on Photos.app)
**Project Type**: CLI tool (single bash script + bats test suite under `scripts/tests/`)
**Performance Goals**: Test suite under 2 minutes (SC-008); full sync run unbounded (background, hours acceptable)
**Constraints**: Idempotent re-runs; local-only (Constitution V); osxphotos CLI is the only path to Photos.app metadata; tests must run on host with the library
**Scale/Scope**: ~80,258 photos + videos in production library; ~6,663 with GPS; ~1,885 favorites in shared scope; ~50 person-named photos; ~unknown user-created albums
